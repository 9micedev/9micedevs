# Security Policy

## Reporting Issues

If you find a security issue, avoid posting exploit details, secrets, tokens, cookies, or account-specific data in public issues.

Report the problem privately to the repository owner when possible, or open a minimal public issue that describes the affected area without sensitive details.

## Responsible Use

This repository is for scripting experiments and utility development. Do not use the code for unauthorized access, cheating, account compromise, credential theft, or disruption of Roblox experiences.

## Secret Handling

- Do not commit tokens, cookies, executor keys, or private configuration.
- Review diffs before pushing changes.
- Rotate any secret that was accidentally committed or shared.
